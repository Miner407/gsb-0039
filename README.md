# 简易工时填报与项目成本统计系统

## 系统用途

本系统是一个轻量级的工时填报与项目成本统计工具，支持团队成员填报工时、管理员审核与统计分析。系统采用纯前端单页应用 + Node.js Express 后端 + SQLite 数据库的架构，部署简单，易于使用。

主要功能：
- 成员端：工时填报、编辑、删除、批量填报、工时复制、个人统计
- 管理员端：工时审核（通过/驳回）、项目管理、用户管理、多维度统计报表、CSV 导出

## 运行环境

- Node.js >= 14.0.0
- npm 或 yarn
- 支持 Windows / macOS / Linux

## 安装依赖

```bash
npm install
```

## 初始化数据库

首次运行前需要初始化数据库并创建默认账号：

```bash
npm run init-db
```

数据库文件将创建在 `data/` 目录下。

## 启动服务

```bash
npm start
```

服务默认运行在 `http://localhost:3001`

端口可通过环境变量 `PORT` 修改：

```bash
PORT=8080 npm start
```

## 成员端功能

1. **工时填报**：选择项目、填写任务名称、工时数、日期和备注
2. **工时编辑**：对待审核和已驳回的工时可以编辑
3. **工时删除**：删除待审核或已驳回的工时记录
4. **批量填报**：一次提交同一天的多条工时记录
5. **工时复制**：将某一天的工时记录复制到另一天
6. **每日汇总**：查看每日工时汇总和列表
7. **个人统计**：查看个人的工时统计数据
8. **审核状态查看**：查看工时的审核状态和审核意见
9. **重新提交**：被驳回的工时可以修改后重新提交审核

## 管理员端功能

1. **工时审核**：审核通过或驳回，填写审核意见
2. **项目管理**：新增、编辑、删除项目，设置项目名称和小时费率
3. **用户管理**：新增、编辑、删除用户，设置用户角色和小时费率
4. **项目成本统计**：按项目汇总工时和成本
5. **成员工时统计**：按成员汇总工时和成本
6. **任务类型统计**：按任务类型汇总工时
7. **项目成员交叉统计**：按项目和成员交叉汇总
8. **未审核工时列表**：查看所有待审核的工时记录
9. **成员利用率统计**：按日期范围计算成员利用率（已审核工时 / 工作日标准工时）
10. **CSV 导出**：导出项目成本和成员工时统计为 CSV 文件

## 接口列表

### 认证接口
- `POST /api/auth/login` - 登录
- `POST /api/auth/logout` - 登出
- `GET /api/auth/me` - 获取当前用户信息

### 项目接口
- `GET /api/projects` - 获取项目列表
- `POST /api/projects` - 新增项目（管理员）
- `PUT /api/projects/:id` - 编辑项目（管理员）
- `DELETE /api/projects/:id` - 删除项目（管理员）

### 工时接口
- `GET /api/timesheets` - 获取工时列表
- `GET /api/timesheets/daily-summary` - 每日工时汇总
- `POST /api/timesheets` - 提交工时
- `POST /api/timesheets/batch` - 批量提交工时
- `POST /api/timesheets/copy` - 复制工时
- `PUT /api/timesheets/:id` - 编辑工时
- `DELETE /api/timesheets/:id` - 删除工时
- `POST /api/timesheets/:id/approve` - 审核通过（管理员）
- `POST /api/timesheets/:id/reject` - 审核驳回（管理员）
- `POST /api/timesheets/:id/resubmit` - 重新提交审核

### 统计接口
- `GET /api/stats/summary` - 数据概览
- `GET /api/stats/by-project` - 按项目统计
- `GET /api/stats/by-member` - 按成员统计
- `GET /api/stats/by-task` - 按任务类型统计
- `GET /api/stats/project-member-cross` - 项目成员交叉统计
- `GET /api/stats/pending` - 未审核工时列表
- `GET /api/stats/utilization` - 成员利用率统计
- `GET /api/stats/export/by-project.csv` - 导出项目成本 CSV
- `GET /api/stats/export/by-member.csv` - 导出成员工时 CSV

### 用户接口（管理员）
- `GET /api/users` - 获取用户列表
- `POST /api/users` - 新增用户
- `PUT /api/users/:id` - 编辑用户
- `DELETE /api/users/:id` - 删除用户

## 数据库文件位置

数据库文件位于项目根目录的 `data/` 文件夹中：
- `data/timesheet.db` - 主数据库
- `data/sessions.db` - 会话数据库

## 已知限制

1. 每日工时上限为 12 小时
2. 标准工作日工时为每天 8 小时（用于利用率计算）
3. 已审核通过的工时记录不能编辑或删除
4. 数据库为文件型数据库，不支持高并发场景
5. 会话存储在 SQLite 中，重启服务后登录状态可能失效
6. 密码使用 bcryptjs 进行哈希存储
7. CSV 导出使用 UTF-8 编码（带 BOM），可直接在 Excel 中打开

## 手动验证步骤

### 1. 初始化与登录
1. 运行 `npm install` 安装依赖
2. 运行 `npm run init-db` 初始化数据库
3. 运行 `npm start` 启动服务
4. 浏览器访问 `http://localhost:3001`
5. 使用 `zhangsan / 123456 登录，验证成员端功能

### 2. 成员端验证
1. 点击"新增工时"，填写信息提交
2. 验证工时列表显示刚提交的记录，状态为"待审核"
3. 点击"批量填报"，提交多条记录
4. 验证批量提交成功，检查每日汇总正确
5. 点击"复制工时"，选择源日期和目标日期
6. 验证复制成功，目标日期有相同的工时记录

### 3. 管理员端验证
1. 退出登录，使用 `admin / admin123` 登录
2. 进入"工时审核"页面
3. 选择一条记录，点击"审核"，选择"驳回"，填写驳回意见
4. 验证记录状态变为"已驳回"，显示审核意见
5. 退出登录，使用成员账号登录
6. 找到被驳回的记录，点击"重新提交"
7. 验证状态变回"待审核"，审核意见清空
8. 再次用管理员登录，审核通过该记录
9. 验证记录状态变为"已通过"，显示审核意见和审核人
10. 验证已审核的记录不能编辑和删除

### 4. 统计报表验证
1. 管理员登录
2. 进入"项目统计"页面，选择日期范围
3. 验证项目成本统计数据正确
4. 点击"导出 CSV"，验证下载的 CSV 文件可在 Excel 中正常打开
5. 进入"成员统计"页面，验证成员工时统计
6. 进入"任务类型"页面，验证任务类型统计
7. 进入"利用率"页面，验证成员利用率统计
8. 进入"待审核"页面，验证未审核工时列表

### 5. 项目和用户管理验证
1. 管理员登录
2. 进入"项目管理"，新增、编辑、删除项目
3. 进入"用户管理"，新增、编辑、删除用户
